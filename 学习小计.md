## 2026-02-11 MySQL 建表踩坑

- 问题：从公众号复制代码导致 `\u00A0` 空格报错
- 解决：用 `cleanSQL()` 清理非法字符或者直接去github里面复制
- 教训：**永远不要直接粘贴富文本代码**，光哥里面用的是 不间断空格（No-break space）Unicode：\u00A0

### 代码示例

```js
function cleanSQL(str) {
  return str.replace(/\u00A0/g, ' ').trim();
}
```

## 2026-02-25 结构化输出

### 结构化输出的方式

1. JsonOutputParser： langchain封装好的工具，在 prompt 里放一段格式的提示词然后对返回的结果按照格式来 parse（缺点：依赖llm必须是纯json，因为其实就是直接用了json.parse()方法）
2. StructuredOutputParser：**增强版JsonOutputParser**。通过 prompt 引导 LLM 输出指定结构，在提示词里面可以看到对输出的约束。最好使用StructuredOutputParser.fromZodSchema来借助zod描述结构。
3. tool:直接用 tool 来获取结构化的结果。使用response.tool_calls[0].args来获取调用该工具时候的入参，该参数就是zod描述的js对象，再使用JSON.stringify(result, null, 2)就可以变成美观、标准、可存储/传输的 JSON 字符串（优点：输出的格式一定是符合要求的）。
4. withStructuredOutput：langchain/openai自带的api，它会判断模型是否支持 tool calls，支持的话就用 tool 的方式获取结构化数据，否则用 output parser 的方式，不用我们自己去处理。

*总结：需要流式输出或者输出格式不是json时只能用OutputParser ；需要结构化输出时用withStructuredOutput最方便，只有在它不可用时，才退而求其次用 StructuredOutputParser。*

额外：JsonOutputToolsParser 能在 AI 流式生成工具调用参数的过程中，第一时间识别出完整的参数，从而立即调用工具、提供即时反馈。而如果只是普通流式（没有这个解析器），即使开了 streaming，也无法提前知道参数何时生成完，仍然要等 LLM 输出全部内容后才能调用工具。
